Every great Roblox game benefits from assets that go beyond the default parts library, and that's exactly why I learned Blender. Creating custom 3D models allows me to give each game project a unique visual identity — whether that means crafting weathered stone ruins for a dark-fantasy RPG, designing sleek sci-fi furniture for a futuristic roleplay, or sculpting whimsical props for a colorful obstacle course world. My modeling workflow is built around Roblox's technical requirements, so every asset I produce is optimized for in-game performance without sacrificing visual quality. From concept to import-ready mesh, I handle the entire pipeline myself.

## Featured Models

<AccordionGroup>
  <Accordion title="Emberfall Stone Ruins Set">
    This collection of crumbling stone architecture was created specifically for the Emberfall RPG map project. The set includes broken wall segments, collapsed archways, moss-covered pillars, and scattered rubble pieces — all designed to fit together naturally to form convincing ancient ruins. The models were built with a hand-sculpted aesthetic in mind, using subtle surface displacement to give the stone a rough, aged texture. Each piece sits between 800 and 2,400 polygons, keeping the combined scene well within Roblox's recommended polygon budget. The set has since been reused in several other dark-fantasy builds.
  </Accordion>
  <Accordion title="Sunrise City Furniture Pack">
    A comprehensive interior furniture pack built to furnish the dozens of buildings inside Sunrise City Roleplay. The pack includes over 30 individual models: sofas, dining tables, office desks, kitchen counters, bookshelves, beds, and bathroom fixtures. Every item was UV-unwrapped and given a clean, slightly stylized texture that reads well at Roblox's typical camera distances. Polygon counts were kept especially low for furniture — ranging from 200 to 900 polygons per piece — since many instances of each model appear simultaneously across the city map. The pack significantly reduced the game's part count compared to using Studio parts for the same furniture.
  </Accordion>
  <Accordion title="NexusHub Tech Terminal">
    A standalone sci-fi control terminal created for the NexusHub shop UI demo environment. The terminal features a sleek angular chassis, a raised holographic display panel, and glowing edge-light details. It was modeled with hard-surface techniques in Blender, using beveled edges and a clean topology flow to ensure smooth in-game rendering. The model comes in at approximately 1,600 polygons and uses a single 512×512 texture atlas covering both the diffuse colors and an emissive channel for the glowing accents. It serves as both a decorative asset and the visual anchor for the in-game shop interface.
  </Accordion>

Each tree was sculpted with a strong, readable silhouette in mind — crucial for assets that appear at long render distances. Polygon counts range from 600 for the simple dead tree to 3,200 for the detailed spirit willow, with LOD-friendly geometry that degrades gracefully at distance. ## Technical Specs

Keeping models optimized for Roblox is a core part of my workflow. Here are the technical standards I follow for every asset I produce:

- **Polygon Count Targets:** General props: 500–2,000 polygons. Architectural pieces: 1,000–4,000 polygons. Character accessories: 300–1,200 polygons. Hero assets (featured centerpieces): up to 6,000 polygons.
- **File Formats:** Models are exported from Blender as **FBX** for Roblox Studio import (preferred for rigged accessories) or **OBJ** for static mesh parts. Both formats are tested before delivery.
- **Texture Sizes:** Standard props use **256×256** or **512×512** textures. Larger architectural assets and hero pieces use **1024×1024**. Textures are atlased wherever possible to reduce draw calls.
- **Roblox Import Process:** Static meshes are imported via Studio's **Asset Manager** as MeshParts. Textures are uploaded separately as Decals or Image assets and applied via SurfaceAppearance for PBR materials where supported. Each import is tested in-engine for scale, pivot alignment, and texture fidelity before being placed in the final scene.

<Tip>
  When creating assets for Roblox, keeping polygon counts as low as possible — while preserving a clean silhouette and key surface details — makes a huge difference in game performance. A scene with hundreds of low-poly props will almost always run smoother than one with a handful of high-poly hero assets. I always ask: "What's the minimum polygon count that still makes this model look great?"
</Tip>
