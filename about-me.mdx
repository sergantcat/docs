## My Journey

My story with Roblox started the way it does for a lot of developers — as a player. I spent countless hours exploring other people's games, and at some point I stopped asking *"how do I play this?"* and started asking *"how do I build this?"* That curiosity pulled me into Roblox Studio, where I began experimenting with parts, terrain, and basic builds. The more I learned, the more I wanted to create, and before long, building had become my favourite thing to do.

After getting comfortable with Studio's built-in tools, I wanted to push the visual quality of my projects further. That's what led me to Blender. Learning 3D modeling was a steep but incredibly rewarding curve — understanding meshes, UV unwrapping, and the art of keeping polygon counts sensible for a game engine. Now I use Blender regularly to create custom assets: stylised props, environment pieces, and decorative details that I couldn't achieve with standard Roblox parts alone. Seeing one of my Blender models imported and placed inside a live game world never gets old.

UI design came next, almost naturally. I noticed that even beautifully built games could feel frustrating if the menus, health bars, and popups were cluttered or confusing. I dived into Roblox Studio's UI system — frames, text labels, image buttons, tweening — and taught myself how to design interfaces that feel smooth and intuitive. Good UI is invisible in the best possible way: players just *know* what to do without thinking about it, and I find that challenge genuinely exciting.

More recently, I've started exploring JavaScript. My goal is to broaden my skills beyond Roblox so I can build web-based tools, portfolio sites, and eventually tie web technologies into my game development workflow. It's early days, but I'm already enjoying how many concepts carry over from scripting in Lua.

## Tools & Technologies

<AccordionGroup>
  <Accordion title="Roblox Studio">
    Roblox Studio is my primary development environment. I use it for level design, terrain sculpting, part-based building, scripting game logic in Lua, and wiring up all the systems that make a game feel complete. I'm comfortable working with tools like the constraint system, surface appearances, and the properties panel to fine-tune every detail of a build.
  </Accordion>
  <Accordion title="Blender 3D Modeling">
    Blender is where I create custom meshes for my Roblox projects. My workflow typically involves box modeling or sculpting a base shape, retopologising for a clean edge flow, UV unwrapping, and then exporting as an `.fbx` or `.obj` file ready for import into Studio. I also use Blender for quick renders when I want to showcase an asset on its own.
  </Accordion>
  <Accordion title="UI Design in Roblox">
    Inside Roblox Studio I design all of my game interfaces using the built-in GUI objects — ScreenGui, Frame, TextLabel, ImageButton, and more. I rely heavily on tweening and animations to give UIs a polished, responsive feel. I also think carefully about layout, colour, and typography so that every interface matches the overall aesthetic of the game it belongs to.
  </Accordion>
  <Accordion title="JavaScript & Web Basics">
    I'm actively learning JavaScript, working through core concepts like DOM manipulation, functions, arrays, and asynchronous programming. I'm also picking up foundational HTML and CSS so I can build and style web pages from scratch — skills that are already coming in handy for building out this very portfolio!
  </Accordion>
</AccordionGroup>
